  • Listado de las Capas propias del host
    • Capa de aplicación

      Interfaz de servicios finales, contiene toda la lógica necesaria para posibilitar las distintas aplicaciones de usuario, las aplicaciones pueden acceder a la capa de transporte por los puertos asignados y los protocolos que tiene son FTP, Telnet, WWW, DNS, ...

    • Capa de transporte (TCP, UDP)

      Transferencia entre proceso de aplicación P2P, la identificación de transportes ser realiza mediante los puertos. Hay dos tipos de protocolos de transporte:

      • TCP (Transport control protocol) - orientado a conexiones y complejo
      • UDP (User datagram protocol) - no orientado a conexión y más sencillo

    • Capa de internet

      Gestiona el movimiento de paquetes por la red, transferencia entre maquinas origen y destino, identificación mediante direcciones IP, no es muy fiable y no está orientada a conexión, las funciones que realiza son el encaminamiento y la fragmentación ,y comunica de los errores con ICMP y SNMP

    • Capa de interfaz de red

      Proporciona los drivers que interactúan con los componentes HW para adaptar y configurar el nivel físico, tiene funciones de nivel físico y enlace.

  • Capa de interfaz de red
    • Necesidad de la capa de interfaz de red

      El uso de red provoca que sea necesario la utilización de niveles intermedios que conecten el IP con las subredes. La interfaz de red actúa como interfaz de conexión entre la red de datos y el computador, cada una de ellas tienen una dirección IP.

    • Datagrama
    • IP sobre Ethernet

      Las tarjetas de red tienen una MAC unica en el HW, el campo longitud/Tipo de la trama MAC muestra la longitud de la trama y el tipo de protocolo:

      • 0800H para IP
      • 0806H para ARP

      Toda máquina que ejecute una aplicación TCP/IP y este escuchando la red comprobara el campo L/T, una estación que ejecute otro protocolo dará la trama anterior como invalida

    • Protocolo ARP
      • Definición

        El protocolo ARP permite la configuración automática de correspondencias de direcciones MAC y direcciones IP. Su funcionamiento es el siguiente :

        El host busca su propia cache ARP local una dirección coincidente para la IP de otro host, en el caso de que no encuentre la correspondencia envía una trama ARP request preguntando por la dirección IP, el otro host sera el único que conteste enviando un paquete ARP indicando dirección IP y dirección Ethernet, las direcciones ser mantendrán cierto tiempo en la cache ARP.

  • Capa IP
    • Direccionamiento IPv4

      Para proporcionar un direccionamiento uniforme en internet, el estándar IP define un esquema de direccionamiento abstract que asigna a cada interfaz de red una dirección de protocolo IP unica. La dirección de internet IPv4 es de 32 bits único, lo cual da un total de 4000 millones de direcciones posibles, el esquema de direccionamiento IP es independiente del MAC, en los paquetes IP las compos de destino y origen están en la cabecera y cada uno contiene 32 bits.

    • Características del esquema de direccionamiento

      Las direcciones IP están divididas jerárquicamente en prefijo y sufijo

      • El prefijo identifica la red y no puede haber dos redes en el mismo prefijo
      • El sufijo identifica el host y no puede haber dos dispositivos con el mismo sufijo
      • Las dir. IP se representan con 4 números y notación punto decimal
    • Direccionamiento Clases IP
    • Direccionamiento sin clase, CIRD
      • Concepto

        Con el crecimiento de internet se comprobó que el direccionamiento con clases era poco flexible y eficiente para ello se creó el direccionamiento sin clases que se basa en el cambio del concepto de clase por el de prefijo de red permitiendo definir bloques de direcciones de cualquier tamaño y así los host utilizaban este prefijo para determinar el punto de division entre número de re y número de host

      • Concepto de mascara
      • Notación CIDR
      • Subredes
        • Definición

          Las redes físicas se pueden dividir en varias redes más pequeñas o subredes tomando bits de la parte del host de la dir. Los routers marcan los límites entre las subredes, definiendo una máscara en la que están a 1 los bits de las red-subred (prefijo extendido) y 0 a los del host. Las subredes mejoran la asignación de dirs. y encaminamiento.

        • Subredes caso de estudio
        • División de subredes de tamaño variable, VLSM
          • Concepto

            La subdivision de la red (Subneteo) de tamaño variable es otra de las soluciones adoptadas para que no sufra de agotamiento de direcciones IP, como el CIDR, NAT y direcciones IP privada. El CIDR divide de forma homogénea los espacios de memoria, pero se desperdician direcciones. La técnica VLSM puede dividir red en subredes de diferentes tamaños, utilizando máscaras de subred de tamaño variable.

            Su funcionamiento es tomar una red y dividir subredes fijas, luego se toma una de las subredes y se vuelve a dividir, tomando bots prestados de la porción de hosts, ajustando a la cantidad de hosts requeridos por cada segmento de la red.

          • Casos de estudio
        • Agregación de rutas - superredes (Supernetting)
          • Concepto

            El problema de lo anterior es la ineficiencia de la asignación de rangos IP y evitar la sobrecarga de las tablas de encaminamiento de los routers. Por lo que las superredes surgen para la prevención, estas redes están formados por la combinación de dos o más redes contiguas con un prefijo CIDR, el prefijo de enrutado la superred comprende los prefijos de las redes que la constituye. La técnica consiste en tomar dos o más bloques de direcciones de red numéricamente contiguas y se consolidan en una sola dirección de red más grande con una máscara más corta

  • Protocolo ICMP

    Los protocolos IP son de transferencia de datos y están acompañados por otros protocolos de uso a nivel de red como ARP, DHCP e ICMP.

    El ICMP, Internet Control Messages Protocol se utiliza para la gestion de mensajes de error y otras condiciones que requieran atención. Estos mensajes se activan por el nivel IP o protocolos de nivel superior (UDP, TCP), son generados por el host o routers que detectan el problema o situación extraordinaria y dirigidos al host o router que aparece en el campo dirección origen del datagrama que causo el problema, estos mensajes viajan como datagramas IP, sujetos a las reglas de los routers como cualquier datagrama y ICMP usa IP como si fuera un protocolo de nivel superior, pero implementados con el módulo IP.

    • Tipos de mensajes
    • Formato
    • Mensajes de error
      • Destination unreachable, destino inalcanzable: tipo 3 - el campo de datos contiene la cabecera IP y los primeros 8 bytes de la carga util del datagrama original. Un router puede emitir este mensaje si no se puede entregar el datagrama en su destino porque el router se encuentra con un datagrama con el bit DF=1 y no cabe en la MTU de la red o el router no se encuentran en ninguna tabla de las rutas por las que pueda llegar a la dirección de destino del datagrama.
      • Source Quench, supresión de origen: tipo 4 - aviso de descarte por congestion, permite a los routers solicitar reducción de tráfico por los hosts en caso de congestion
      • Time exceeded for datagram, tipo 11 - aviso de un descarte por exceso de tiempo en el sistema (TLL=0) no se usan los cuatro bytes finales de la cabecera, el campo de datos contiene la cabecera IP y los 8 primeros B de la carga util del datagrama original.
      • Redirect, tipo 5 - se utiliza para alertar al host emisor cuando sospecha que un paquete este mas encaminado, los 4bytes finales de la cabecera contienen la dirección del router, el campo dato contiene la cab IP y los primero 8 byte de la carga util del datagrama original

  • Encaminamiento
    • Definición encaminamiento IP

      El encaminamiento IP es el proceso de reenviar un paquete basado en la dirección de destino. Los métodos de encaminamiento son:

      • Estático o no adaptativo - no tiene en cuenta el estado de la red, existe una ruta permanente origen-destino
      • Dinámico o adaptativo - las decisiones cambian a medida de las condiciones, creadas y mantenidas automáticamente por protocolos de routing
    • Concepto de ruta

      Se establece gracias a las diferentes tablas de encaminamiento de los distintos dispositivos (routers) que intervienen en la comunicación (a nivel IP). Tipos de rutas :

      • Directas - host a host se comunican entre si sin necesidad de un dispositivo intermedio de la capa de red
      • Indirectas - a traves de uno o más routers se establece salto a salto
